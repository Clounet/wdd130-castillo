Client Report

How did you decide what content and images to use for your site plan?

As the client for the Underrated website, I wanted the site to focus on Stephen Curry's journey, achievements, and influence on basketball. I selected content that explained how he overcame doubts and became one of the best shooters in basketball history.

I chose images that were connected to Stephen Curry, basketball, his career, and his accomplishments. I wanted the images to make the site more interesting and help visitors understand the theme of being underestimated and overcoming challenges.

How did you communicate with your developer? Were there any issues with communication?

I communicated with my developer by sharing the purpose of the site, the content I wanted, and ideas for the images and design. I also reviewed the published website and gave feedback about the pages and visual appearance.

Communication was generally good. We were able to discuss the project and confirm the main requirements. One challenge was making sure that all the details were completed on time, but reviewing the published website helped us see what was finished and what still needed improvement.

Did your developer meet your expectations for your site?

Yes, my developer met my main expectations. The website has a clear topic, organized information, relevant images, and working navigation. The design communicates the idea of Stephen Curry being underrated and shows his development, accomplishments, and legacy.

I was satisfied with the final result because the website was published successfully and visitors can move between the pages and understand the purpose of the site.

What will you do differently next time you hire a team to do a project for you?

Next time, I would give the developer more detailed written instructions at the beginning. I would also create smaller deadlines for the site plan, content, images, HTML, CSS, and final review.

I would check the progress more frequently so that problems could be identified earlier. This would give the developer more time to apply feedback before the final deadline.

Lead Developer Report

How did you plan and coordinate the website?

As part of the developer team for World Cup Central, I helped follow the site plan and maintain a consistent design across the website. The pages needed to use similar colors, fonts, navigation, spacing, images, and content organization.

I reviewed the existing HTML structure and the shared CSS file before working on my assigned page. This helped me understand the design that the team was using and prevented my page from looking different from the rest of the site.

How did you organize the development workflow?

The work was divided into different pages and responsibilities. I focused on the World Cup History page while also checking that the shared navigation and visual style worked correctly.

I first organized the HTML content, added headings, paragraphs, links, and images, and then used the shared CSS file to improve the visual design. After making changes, I checked the page in the browser and reviewed the published version through GitHub Pages.

How did you solve problems during development?

When a page did not look correct, I reviewed the HTML structure, CSS selectors, image paths, and file names. I also compared the page with the other pages to make sure the layout and navigation were consistent.

Testing the published site helped identify problems that were not always visible in the code editor. I corrected spacing, alignment, readability, and file-path issues when necessary.

Junior Developer Report

How did you communicate with your client? Were there any issues?

I communicated with the client about the World Cup Central website, including the content, images, structure, and appearance of the site. I asked for feedback after the changes were published so the client could review the finished work.

There were no major communication problems, but coordinating the work and completing all the details required attention. It was important to explain what had been completed and confirm whether the client wanted additional changes.

What work did you complete as a junior developer?

My main responsibility was working on the World Cup History page. I helped organize information about the history of the tournament, previous champions, and important World Cup moments.

I used semantic HTML elements, headings, paragraphs, images, and links to create an organized page. I also applied CSS styles so the page matched the colors, fonts, spacing, and layout used throughout the World Cup Central website.

How did you apply feedback and collaborate with the team?

I reviewed the page after publishing it and compared it with the other pages on the site. I made adjustments to improve the spacing, readability, image sizes, and consistency of the navigation.

I followed the shared design instead of creating completely different styles for my page. This helped the website feel like one complete project instead of several unrelated pages.

How did you handle publishing the client's site and sharing it?

The website files were uploaded and committed to the GitHub repository. After the changes were published with GitHub Pages, I opened the live URL and checked that the text, images, navigation, and CSS loaded correctly.

I shared the published link with the client so the site could be reviewed. Checking the live page was important because it confirmed that the website worked outside of the local development environment.

How was your experience managing the site with GitHub?

Managing the site with GitHub was a useful experience. I practiced organizing files, updating code, committing changes, and checking the GitHub Pages website.

I learned that file names and paths must be exact because GitHub Pages can be sensitive to capital letters and folder names. I also learned that published changes may take a short time to appear, so it is important to refresh the site and verify the final result.

HTML and CSS Skill Development

This project helped me improve my understanding of HTML structure and CSS styling. I practiced using headers, navigation elements, sections, headings, paragraphs, links, images, and footers to organize a complete website.

I also practiced using CSS to control colors, fonts, spacing, alignment, image size, navigation, and page layout. I learned that a good HTML structure makes CSS easier to apply and helps keep the code organized.

Another important skill was maintaining a consistent design across multiple pages. Using the same stylesheet and similar HTML structures made the website easier to navigate and gave it a more professional appearance.

General Experience and Reflections

What was the most challenging part of this project? Were you able to overcome it?

The most challenging part was coordinating the design and content across different pages and team roles. Each page had different information, but all the pages needed to look like they belonged to the same website.

I was able to overcome this challenge by using a shared stylesheet, following the same navigation structure, checking the published pages, and correcting inconsistencies in spacing, images, and layout.

What did you enjoy most about this project?

I enjoyed seeing the website develop from a basic HTML structure into a complete visual project. Adding colors, images, fonts, spacing, and layouts made the website more attractive and easier to use.

I also enjoyed publishing the final result online. Seeing the website working through GitHub Pages made the project feel more realistic and gave me experience with a tool commonly used by developers.

What would you do differently if you could do the project again?

If I completed the project again, I would create clearer deadlines and responsibilities from the beginning. I would also communicate more frequently with the client and developers instead of waiting until the final stages to review the work.

I would make smaller GitHub commits, test the pages on different screen sizes earlier, and review the shared CSS more often to prevent inconsistent designs.

Do you feel more confident working on real-world projects after this experience?

Yes, I feel more confident working on real-world projects after this experience. I practiced communicating with a client, understanding project requirements, collaborating with other developers, applying feedback, organizing files, and publishing a website.

I learned that web development is not only about writing HTML and CSS. It also requires planning, communication, testing, problem-solving, and responsibility. This experience helped me understand how a small web development team can work together to complete a project.
